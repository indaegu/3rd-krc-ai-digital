# design-system.md — 디자인 시스템

> UI 작업 전에 읽는다. **이 문서가 시각 토큰·컴포넌트 규칙의 SSOT**다.
> `prototype/mulsigye-app-prototype-v2.html`은 인터랙티브 참고물이며 충돌하면 이 문서를 따른 뒤
> 같은 작업에서 프로토타입을 갱신한다. 제품 행동·수치 의미는 [product.md](product.md)가 우선한다.

## 방향

흰 배경, 단색 카드, 큰 볼드 타이포, 넉넉한 여백, 짧은 ~해요체를 사용한다.
고령 농업인이 햇빛 아래 휴대전화에서 읽고 누를 수 있는지를 우선한다.

## 토큰

```css
--ink:#191F28; --ink2:#4E5968; --ink3:#8B95A1; --ink4:#B0B8C1;
--bg:#FFFFFF; --gray50:#F9FAFB; --gray100:#F2F4F6; --gray200:#E5E8EB;
--blue:#3182F6; --blue-deep:#1B64DA; --blue-tint:#E8F3FF; --blue-soft:#D6E8FF;
/* 공식 가뭄 단계 fg/bg */
--ok-fg:#159570;    --ok-bg:#E6F6F0;
--watch-fg:#9A6700; --watch-bg:#FFF3D6;
--care-fg:#D9510C;  --care-bg:#FFEBDE;
--alert-fg:#E5372F; --alert-bg:#FDEBEA;
--crit-fg:#A11C1C;  --crit-bg:#F8E2E2;
--r-lg:24px; --r-md:18px; --r-sm:12px;
```

- 웹은 전역 CSS 변수, Android는 의미가 같은 Compose `ColorScheme`·shape 토큰으로 옮긴다.
- 이름을 번역 구현하더라도 `ok/watch/care/alert/crit` 의미와 값은 바꾸지 않는다.
- Pretendard Variable을 self-host하며 시스템 고딕으로 폴백한다. 명조·원격 CDN 폰트는 쓰지 않는다.
- 공식 단계 임계값은 색 문서가 아니라 `apps/web/src/lib/data/drought-stage.ts`에만 둔다.

## 레이아웃·컴포넌트

- 메인 모듈 간격 24px. 카드 `--gray50`, radius 24px, 내부 패딩 20px.
- 주 CTA 높이 56px, radius 16px, `--blue`, 글자 굵기 700. 비활성은 `--gray200`.
- 비이커는 대표 저수지 원저수율만 보여주며 가뭄 단계 눈금을 겹치지 않는다. 단계 임계는
  지역 `avgRatio` 기준이라 원저수율 높이와 같은 축이 아니다.
- 단계 칩은 단색 tint와 **“지역 평년 대비 기준”** 보조 라벨을 함께 쓴다.
- 메인 게이지 제목은 **“우리 지역 대표 저수지”**, 값 라벨은 **“현재 저수율”**로 쓴다.
- 그래프 제목·축·대체 텍스트에는 **“지역 평년 대비 저수율”**을 명시한다.
- 단계 뱃지와 만수위 참고 배너를 같은 위험 체계처럼 보이게 합치지 않는다.
- 바텀시트는 상단 radius 24px, 그랩바, `rgba(25,31,40,.45)` 딤을 쓴다.

## 로딩 패턴

| 상황 | 패턴 |
|---|---|
| 메인 진입·새로고침 | 모듈별 스켈레톤(shimmer 1.3s); 풀스크린 스피너 금지 |
| 주소 검색 | 결과 영역 인라인 스피너 |
| 등록·삭제 버튼 | 버튼 내부 흰 스피너 + 중복 입력 잠금 |
| 헤더 기준 시각 | “불러오는 중…” 텍스트 |

## 애니메이션

- 물 출렁임: 회전 타원 2겹(7s / 11s reverse), 수위 0→목표 1.6s.
- 로고 새로고침: 물방울 rainfall 0.62s. 로고 회전 금지.
- 화면 전환: fade + 7% 슬라이드, 0.3s.
- 웹 `prefers-reduced-motion`과 Android 시스템 “애니메이션 삭제” 설정에서 장식 모션을 끈다.
- 새 애니메이션은 두 플랫폼의 reduced-motion 처리와 함께 추가한다.

## 접근성

- 본문 15px 이상, 핵심 숫자 3rem급, 텍스트 대비 WCAG AA 이상.
- 터치 목표 최소 48×48dp, 아이콘 단독 버튼에는 접근 가능한 이름을 제공한다.
- 웹은 `:focus-visible` 아웃라인, 의미 있는 heading 순서, 라이브 영역 남용 금지.
- Android는 TalkBack 읽기 순서·contentDescription·큰 글꼴 1.3배에서 레이아웃을 확인한다.
- 색만으로 단계를 구분하지 않고 단계명과 기준을 텍스트로 쓴다.

## 콘텐츠 가드

- “가까운 저수지”, 거리, 알림 CTA, 로그인 CTA를 만들지 않는다.
- 예측은 “가능성이 있어요” 형식만 사용하고 공식 예·경보 우선 문구를 함께 둔다.
- 오류는 원인을 쉬운 말로 설명하고 재시도 가능할 때만 재시도 버튼을 표시한다.

## 갱신 규칙

토큰·컴포넌트·플랫폼 매핑을 바꾸면 웹 CSS, Compose Theme, 프로토타입을 같은 변경에서 맞춘다.
프로토타입에서 먼저 실험했더라도 승인된 결과를 이 문서에 반영해야 제품 규칙이 된다.
