# data-sources.md — 공공데이터 명세

> 데이터 페치·가공 코드를 쓰기 전에 읽는다. 모든 출처는 공공데이터포털(data.go.kr)의
> 한국농어촌공사 개방 데이터다. **공모전 규정상 이 데이터 사용은 필수 요건이다.**

## 사용 데이터 4종

### 1) 농촌용수 저수지 수위정보 조회 (API) — 실시간 축 ⭐

- 형태: REST API (개방 목록 #95) · 활용신청 자동승인
- 역할: 선택 저수지의 날짜별 수위·저수율 → 메인 게이지·그래프의 원천
- **호출 제약(중요)**: 저수지명 기준 최대 365일 / 위치(좌표) 기준 최대 31일 조회
- 인증: `DATA_GO_KR_API_KEY` (.env, 커밋 금지)
- 장애 대비: 60분 캐시 + 마지막 성공 응답 폴백 (architecture.md 참조)

### 2) 전국 저수지 일별 저수율 (CSV) — 학습·백테스트 축

- 형태: 파일데이터(CSV), 시군구별 일별 저수율·유효저수량 (약 3,400+ 행 규모)
- 역할: 추세 모델 백테스트, 지역(시군) 단위 보조 지표
- 처리: `scripts/build-data.*` 로 정규화 → `data/reservoir-daily.json` 커밋

### 3) 논가뭄지도 (CSV) — 공인 단계 기준 축

- 형태: 파일데이터, 시군 통합저수율·평년값·평년대비·가뭄단계
- 역할: **가뭄단계 판정의 유일한 근거.** 화면의 관심/주의/경계/심각은 전부 여기서 나온다.

### 4) 가뭄예경보자료 (CSV) — 공식 전망 축

- 형태: 파일데이터, 현재 단계 + 향후 1·2·3개월 전망
- 역할: 자체 예측 옆에 **공식 전망을 병기**해 신뢰를 만든다.
  (자체 예측 단독 노출 금지 — prediction-model.md 참조)

## 가뭄단계 공인 기준 (하드코딩 상수의 단일 출처)

| 단계 | 기준 (평년 대비 저수율) | UI 색 토큰 |
|---|---|---|
| 정상 | 70% 초과 | `ok` |
| 관심 | 70% 이하 | `watch` |
| 주의 | 60% 이하 | `care` |
| 경계 | 50% 이하 | `alert` |
| 심각 | 40% 이하 | `crit` |

- 코드에서는 `lib/data/drought-stage.ts` 한 파일에만 이 임계값을 둔다. 다른 곳에 복제 금지.
- 게이지 눈금(70/60/50/40)은 "평년 대비 기준을 저수율로 환산한 예시"임을 각주로 표시.

## 필드 정규화 규칙

- 외부 필드명(한글/축약 혼재)은 `lib/data`에서 내부 표준으로 변환 후 사용:
  `date`(ISO), `rate`(저수율 %, number), `avgRatio`(평년대비 %, number), `stage`(단계 enum),
  `reservoirCode`, `reservoirName`, `sigunCode`.
- 결측·이상치: 음수/100 초과 저수율은 결측 처리, 연속 결측은 선형 보간(3일 이하만),
  그 이상은 해당 구간 예측 신뢰도 하향 표시.

## 주소 → 저수지 매칭

- 도로명주소 검색: 행정안전부 주소 API(juso.go.kr) 사용 예정 — 키 별도 발급.
- 매칭 규칙 v1: 주소의 시군구 → 해당 시군 대표 저수지(수혜면적 최대) 자동 선택,
  거리 표시는 좌표 기반 직선거리. 정밀 유역 매칭은 범위 외.
