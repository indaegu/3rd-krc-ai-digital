# prediction-model.md — 예측 모델과 백테스트

> 예측·백테스트 코드를 쓰기 전에 읽는다. 이 수치들은 기획서 'AI 활용성' 섹션의
> 근거가 되므로, **결과 재현 가능성**이 정확도만큼 중요하다.

## 목표

최근 저수율 추세로 앞으로 14일을 예측하고,
**"현재 추세 유지 시 다음 가뭄 단계 도달까지 남은 일수"** 를 계산한다.

## 후보 모델 (전부 구현하고 백테스트로 선정)

| 모델 | 정의 | 역할 |
|---|---|---|
| naive | 마지막 값 유지 | 기준선 1 |
| ma7 | 최근 7일 평균 유지 | 기준선 2 |
| linear | 최근 N일 선형회귀 외삽 | 주력 후보 |
| ses | 지수평활(단순) | 주력 후보 |

- 복잡한 딥러닝 금지(v1). 심사 방어력은 "비교 실험 + 수치"에서 나온다.
- 구현 위치: `lib/prediction/models.ts` — 순수 함수, 입력은 `number[]`(일별 저수율).

## 백테스트 프로토콜 (재현 명령: `pnpm backtest` — 스캐폴드 후 확정)

1. 대상: 전국 일별 저수율에서 표본 저수지/시군 K개(대표 3개 지역 포함).
2. 홀드아웃: 시계열 끝 7일·14일을 숨기고 나머지로 예측.
3. 지표: **MAE(%p)** 를 주지표, RMSE 보조. 모델×홀드아웃 조합별 표 산출.
4. 선정: 14일 홀드아웃 MAE 최저 모델을 서비스 기본으로 채택.
5. 산출물: `data/backtest-report.json` + 요약을 이 문서 하단 '결과' 절에 기록.

윈도(N), 평활계수 등 하이퍼파라미터는 코드 상수로 두고 값 변경 시 리포트 재생성.

## 도달 예상일 계산

```
현재 평년대비 r0, 일일 변화율 d(선정 모델 기준)
다음 단계 임계 t (70/60/50/40 중 현 단계 바로 아래)
도달일 = ceil((r0 - t) / |d|)   단, d ≥ 0 이면 "안정"
```

- 표시 규칙: 1~30일 → "N일 뒤", 31일 초과 또는 안정 → "안정" 카드.
- 도달일과 함께 **항상** 예측 오차를 병기: "예측 오차(최근 14일) ±X.X%p".

## 불확실성·표현 규칙 (절대 준수 — AGENTS.md 규칙 3)

- 문장 템플릿: "지금 속도면 N일 뒤 '단계'로 내려가요" / "~할 것으로 보여요".
- 그래프: 실측=실선, 예측=점선, 불확실 구간=밴드. 세 요소 분리 표기는 필수.
- 화면 어딘가에 항상: "예측은 참고용이며 공식 가뭄 예·경보가 우선" 고지.
- 가뭄예경보의 공식 1~3개월 전망을 자체 예측과 **병기**한다(단독 노출 금지).

## 만수위 참고 로직 (부가 기능)

저수율 ≥ 95% 이고 추세 d > 0 이면 파란 '참고' 배너 + 배수로 점검 행동 세트로 전환.
'경보' 어휘 금지 (product.md 참조).

## LLM 행동요령 생성 (coach)

- 입력: 지역명, 현 단계, 도달 예상, 계절 → 출력: 인사 1문장 + 행동 3개(굵은 줄+보조 줄).
- 프롬프트에 단계별 허용 행동 목록을 제약으로 제공(자유 생성 금지) — 안전한 범위 유지.
- LLM 장애 시 폴백: 단계별 정적 행동 세트(프로토타입의 문구)를 그대로 사용.
