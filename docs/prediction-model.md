# prediction-model.md — 예측 모델과 백테스트

> 예측·백테스트 전에 읽는다. 기획서의 AI 활용 근거이므로 정확도뿐 아니라
> 데이터 누수 방지, 재현 명령, 결과 보존이 필수다.

## 예측 목표와 입력

- 목표: 지역의 일별 **평년 대비 저수율 `avgRatio`**를 앞으로 14일 예측한다.
- 입력: 논가뭄지도의 한 시군 `avgRatio` 시계열. 대표 저수지 원저수율 `rate`를 넣지 않는다.
- 부가 출력: 현재 하락 추세가 유지될 때 다음 공식 단계에 이를 가능성이 있는 일수(최대 30일).
- 공식 단계·전망을 대체하지 않는다. 자체 결과는 항상 참고 정보로 표시한다.

## 후보 모델

모두 `apps/web/src/lib/prediction/models.ts`의 결정적인 TypeScript 순수 함수로 구현한다.

| 모델 | 정의 | 역할 |
|---|---|---|
| `naive` | 마지막 값 유지 | 기준선 1 |
| `ma7` | 최근 7일 평균 유지 | 기준선 2 |
| `linear` | 최근 N일 선형회귀 외삽 | 주력 후보 |
| `ses` | 단순 지수평활 | 주력 후보 |

- 입력은 `number[]`(날짜 오름차순의 `avgRatio`), 출력은 14일 예측 `number[]`다.
  입력이 14일 미만이면 명시적 에러를 낸다(모델별 최소 길이 상수, 전 모델 14일 통일).
- 복잡한 딥러닝은 v1 범위 밖이다. 기준선과 비교한 수치로 모델 선택을 방어한다.
- N·평활계수는 이름 있는 상수와 모델 버전에 포함한다. 변경하면 리포트를 다시 만든다.
- 확정 상수(단계 3 플랜 승인): `LINEAR_WINDOW_DAYS = 14`, `MA_WINDOW_DAYS = 7`,
  `SES_ALPHA = 0.3`, `MODEL_VERSION = "pred-v1"`.
- “더 단순한 모델” 동률 판정 서열(`MODEL_SIMPLICITY_ORDER`, 앞이 더 단순):
  `naive < ma7 < ses < linear`.

## 백테스트 프로토콜

재현 명령은 스캐폴드 후 `pnpm backtest`로 고정한다.

1. 논가뭄지도에서 유효 관측 180일 이상인 모든 시군 시계열을 사용한다.
2. 각 시계열 마지막 90일 안에서 14일 간격의 rolling origin을 만들며 미래값을 학습에 넣지 않는다.
3. 각 origin에서 7일·14일을 예측해 모델별 MAE(%p)와 RMSE(%p)를 계산한다.
4. 지역별 지표와 지역별 MAE의 단순 평균(macro average)을 모두 낸다.
5. 기본 모델은 **14일 macro MAE 최저**로 선택한다. 차이가 0.05%p 이하면 더 단순한 모델을 고른다.
6. 관측 수 부족·긴 결측 구간은 제외하고 지역 코드·제외 이유를 리포트에 남긴다.
7. `data/backtest-report.json`에 원천 체크섬, 실행 시각, git commit, 모델 파라미터,
   표본·origin 수, 전체 지표를 기록한다.

구현 확정(단계 3 플랜 승인): origin은 `lastDate − 14·k (k=1..6)`로 만들어 마지막
90일 안에서 14일 간격을 이루고, 각 origin 뒤 14일 평가 구간이 시계열 끝을 넘지 않는다.
7일 지표는 horizon 1~7, 14일 지표는 horizon 1~14 잔차(실측−예측)의 MAE/RMSE다.
제외 규칙: 유효 관측 180일 미만은 `insufficient_days`, 연속 결측 7일 초과
(`MAX_GAP_DAYS = 7`)는 `long_gap`. 모든 지표는 소수 4자리로 반올림해 기록한다
(재실행 재현성 비교의 기준). 엔진은 `apps/web/src/lib/prediction/backtest.ts`의
순수 함수이고 실행 시각·git commit은 CLI(`apps/web/scripts/backtest.ts`)가 주입한다.
리포트 형태는 `apps/web/src/lib/prediction/backtest-report.ts`의 Zod 스키마가 단일 출처다.

서비스 출시 전 이 문서의 “결과” 절에 리포트 요약과 채택 근거를 작성한다.

## 다음 단계 도달 가능 시점

현재 공식 단계의 바로 아래 임계값만 계산한다.

```
r0 = 최신 avgRatio
d  = 선택 모델이 암시하는 일일 변화량(%p/day)
t  = 다음 단계 임계값(70, 60, 50, 40)

d < 0 이고 r0 > t 이면 days = ceil((r0 - t) / abs(d))
그 외에는 30일 안 도달 가능성 없음
```

- `d`의 통일 정의: `d = (forecast[13] − r0) / 14` (%p/day, 모델 독립적 —
  naive는 자연히 0이다).
- 추세 버킷: `|d| < 0.05`(`TREND_STABLE_EPSILON`)이면 `stable`, 그 외 음수는
  `falling`, 양수는 `rising`. 경계값 ±0.05는 방향을 유지한다.
- 도달 버킷: `days` 없음 → `none`, ≤7 → `within_7d`, ≤14 → `within_14d`,
  ≤30 → `within_30d`.
- 정상→관심 70, 관심→주의 60, 주의→경계 50, 경계→심각 40을 사용한다.
  임계값은 `apps/web/src/lib/data/drought-stage.ts`만 import한다(복제 금지).
- 이미 심각이면 다음 단계 도달일을 계산하지 않는다.
- `days`가 1~30일일 때만 숫자를 표시한다. 그 밖에는 “안정”과 보조 문구
  “현재 추세로는 30일 안에 다음 단계 가능성이 낮아요”를 표시한다.
- 예: `r0=68`, `d=-0.45`, `t=60`이면 `ceil(8/0.45)=18일`이다.
- 예: `r0=46`, `d=-0.67`, `t=40`이면 `ceil(6/0.67)=9일`이다.

## 불확실성·표현

- 실측=실선, 14일 예측=점선, 불확실 구간=밴드로 구분한다.
- 백테스트 잔차가 충분하면 horizon별 경험적 10~90 분위수를 밴드로 쓴다.
  부족하면 최근 14일 MAE를 “예측 오차 ±X.X%p”로 표시하고 밴드 산식을 API 메타데이터에 남긴다.
- 숫자 문구: **“지금 추세가 이어지면 N일 뒤 ‘단계’에 들어설 가능성이 있어요.”**
- 확정 표현(“내려가요”, “됩니다”, “위험합니다”)은 사용하지 않는다.
- 모든 예측 카드·상세에 “예측은 참고용이며 공식 가뭄 예·경보가 우선이에요.”를 표시한다.
- 공식 1·2·3개월 전망에는 “한국농어촌공사 공식 전망”과 발행일을 붙여 자체 예측과 구별한다.

## 만수위 참고 로직

대표 저수지의 원저수율 `rate >= 95`이고 최근 원저수율 추세가 상승할 때만 파란 “참고”
배너와 배수로 점검 행동을 표시한다. 지역 `avgRatio`로 이 조건을 판단하지 않는다.
독자적인 홍수 단계나 경보를 만들지 않는다.

## LLM 행동 코치

정량 예측과 자연어 설명은 분리한다. 이 문서의 순수 함수가 `avgRatio`, 공인 단계,
도달 가능 시점과 오차를 계산하고, LLM은 그 값을 계산·수정하지 않는다. 서버가 선택한
행동 ID·개수·순서를 보존한 쉬운 이유만 생성한다. 상세 호출·캐시·비용·평가 규칙은
`docs/llm-coach.md`를 따른다.
